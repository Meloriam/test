## Создание курса

1.       В панели управления переходим в раздел курсы. Нажимаем кнопку справа сверху «Новый курс +».  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image007.jpg) 

2.       В новом окне выбираем «Начать с нуля».  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image009.jpg)

3.       Заполняем название и описание курса. Указываем видимость «Приватный» для доступа только авторизованным пользователям.  
При желании можно загрузить обложку курса (изображение в соотношении 16:9) и добавить описание чему обучает курс и теги.  
Подтверждаем создание по кнопке «Создать курс».  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image011.jpg) 

4.       По умолчанию курс создается в статусе «Не опубликован» и будет доступен только через панель управления. Для отображения курса на платформе необходимо будет нажать на данную кнопку для его публикации.  
У Активных участников курса в соответствующем разделе курс будет отображаться на главной странице до его публикации.

![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image013.jpg)

5.       Доступ курса. Устанавливается согласно выбору публичности курса при его создании. При необходимости его можно изменить.  
Для того чтобы курс после публикации был виден только определенной части пользователей (например, только для ДЛП) необходимо нажать на кнопку «Связать с группой» и выбрать нужную группу.  
Если же требуется сделать так, чтобы курс был виден всем на платформе (даже без учетной записи), но контент был доступен только некоторым, то можно указать публичный доступ и изменить настройки отдельных глав в разделе «Контент».  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image015.jpg)  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image017.jpg)

6.       Участники курса — список пользователей, которые могут изменять наполнение курса.  
Всего предусмотрено 4 роли:

a.       Создатель — назначается пользователю, создавшему курс. Не изменяется и не переназначается другому.

b.       Сопровождающий — модератор курса с широкими правами доступа для изменения контента.

c.       Репортер — роль, позволяющая оценивать задания, но не дающая прав к редактированию курса.

d.       Участник — роль, которая позволяет изменять контент курса, если после одобрения участник Активен и указано «Открыто для участников».

![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image019.jpg)

## Создание контента курса

1.       Для добавления контента необходимо перейти в соответствующий раздел и нажать «Добавить главу».  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image021.jpg)

2.       Далее система попросит заполнить название и описание главы (оба обязательны к заполнению).  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image023.jpg)

3.       После этого появится блок данной главы. При наличии нескольких глав такие блоки можно перемещать между собой.  
Также по значку замочка можно изменить доступ к материалам курса:

a.       Публичный — позволит любому просматривать материалы курса. Если доступ курса указан «Публичный», нет связи с группами, то все незарегистрированные пользователи будут видеть курс и его контент.

b.       Только авторизованные — контент будет доступен пользователям с учетными записями.

c.       Только группы — контент будет доступен только определенной группе.

![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image025.jpg)

4.       По кнопке «Добавить занятие» появится окно, в котором можно выбрать тип занятия.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image027.jpg)

a.       Динамическая страница — страница, преимущественно для добавления лекционного материала. Кроме текста, картинок и видео позволяет добавить таблицы, аудио, pdf-документы, блоки с уравнениями, интерактивная викторина, внешний объект (перечень поддерживаемых сервисов тут [https://www.learnhouse.app/ru/integrations](https://www.learnhouse.app/ru/integrations)), кнопка (работает только со ссылками на страницы в интернет, не отправляет в сетевой диск), пользователи на платформе.  
По кнопке сохранения записывается версия динамической страницы (до 20 версий).

b.       Видео — видео без дополнительных материалов в формате MP4. Необходимо указать имя. При необходимости можно настроить время начала и конца воспроизведения, а также автоматическое проигрывание и режим без звука при включении.

c.       Документ — PDF-документ без дополнительных материалов. Предпросмотр аналогичен тому, что доступен при просмотре PDF-документов в браузере.

d.       Задания — блоки, в которые позже отдельно добавляются вопросы тестов. При создании задается срок сдачи, тип оценки, а также дополнительные параметры оценивания (автоматическое оценивание, блокировка копирования и вставки, просмотр правильных ответов после проверки, возможность перепрохождения).

e.       Markdown — ссылка на файл в формате markdown.

f.        Внешний встроенный объект (перечень поддерживаемых сервисов тут [https://www.learnhouse.app/ru/integrations](https://www.learnhouse.app/ru/integrations)).

## Блок контента курса: Задание

1.       После создания блока задания его параметры можно при необходимости изменить по кнопке «Редактировать».  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image029.jpg)

2.       В первом разделе «Редактор» отображаются блоки с вопросами теста.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image031.jpg)

3.       У каждого вопроса есть два раздела:

a.       General:

                                                               i.      Заголовок — название вопроса в блоке. Видно только редактору.

                                                             ii.      Описание — отображается под заголовком в блоке, а также на странице самого теста напротив «Задача 1: (тест описания)».

                                                           iii.      Подсказка — всплывающий текст в верхней части страницы. Без заполнения будет всплывать только синий значок.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image033.jpg)

                                                           iv.      Справочный файл — файл, который можно скачать напротив определенного вопроса. **После загрузки файла убрать его нельзя, только заменить на другой файл.**

![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image035.jpg)  
**Перед переходом в раздел** **Content** **необходимо сохранить изменения раздела** **General** **по кнопке «Отправить».**

b.       Content:

                                                               i.      Детали вопроса теста.

![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image037.jpg)

**Все изменения в разделе** **Content** **сохраняются по пунктирной кнопке «Сохранить».**

![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image039.jpg)

c.       Тест — блок, в котором создается вопрос с ответами. Может быть добавлено несколько вопросов с ответами. Дополнительно настраивается несколько ответов к каждому вопросу.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image041.jpg)

d.       Отправка файла — отправка документа на проверку с последующим ручным оцениванием.  
Поддерживаемые форматы файлов: pdf, docx, mp4, jpg, jpeg, png, pptx, zip.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image043.jpg)

e.       Форма — вопрос с текстовым ответом. **Только этот вопрос определяет попытку вставить скопированный текст.**  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image045.jpg)

f.        Код — вопрос, в котором можно написать код программы и проверить его.

g.       Короткий ответ — форма вопроса с текстовым вопросом. **Позволяет вставить ответ, запрет на копирование для данного вопроса не срабатывает.**  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image047.jpg)

h.       Числовой ответ — вопрос с ответом-числом. Позволяет задавать допуск числового ответа +-.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image049.jpg)

4.       В разделе «Работы» перечислены результаты прохождения теста.  
 ![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image051.jpg)  
При «Оценивании» задания необходимо нажимать «запустить автопроверку» напротив каждого вопроса, затем «установить итоговую оценку» и «Завершить и подтвердить» для сохранения оценивания.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image053.jpg)

5.       В разделе «Аналитика» собирается статистка по результатам прохождения после оценивания.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image055.jpg)

# Сообщества

1.       Сообщества представляют собой отдельные блоки, в которых можно собирать идеи и вопросы, а также писать к ним ответы. Они поддерживают только текстовый формат.

2.       Создание сообществ осуществляется через панель управления по кнопке «+ Новое сообщество».  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image057.jpg)

3.       Необходимо указать название. При желании можно добавить короткое описание, а также отметить будет ли сообщество публичным (доступное всем без авторизации) или для определенной группы пользователей.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image059.jpg)

4.       Настройки сообщества настраиваются через панель управления.  
Название, описание, обложка, доступ, связанные курсы (будут отображаться на странице сообщества), модерация.  
В модерации можно добавить ограничения на внешние ссылки, ограничения длины поста и ответа, ограничения времени и количества вопросов-постов, а также запрещенные слова и требования к возрасту аккаунтов.

5.       Доступ к содержимому сообщества осуществляется с главной страницы из соответствующего раздела.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image061.jpg)

6.       Изменение текста обсуждения и его тегов **доступно только его создателю**.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image063.jpg)

Администратор и модераторы могут только закрепить, заблокировать (закрыть для новых ответов) или удалить обсуждение.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image065.jpg)

7.       Поиск по темам обсуждения работает на по всей платформе. По ответам внутри обсуждения поиск не срабатывает.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image067.jpg)

# Доски

1.       Интерактивные доски доступны только авторизованным пользователям.

2.       Создание осуществляется через панель управления.  
Заполняется минимум информации: название, описание, обложка, доступ (для всех авторизированных пользователей или для групп), участники (до 10 пользователей, управляющих доской).

![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image069.jpg)

3.       На главной странице раздел с досками находится по следующему значку.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image071.jpg)

4.       На интерактивной доске можно рисовать, добавлять карточки с текстом, рамки (карточки сами будут следовать за перемещенной рамкой или подстраиваться в нее), добавлять заметки, списки задач, стикеры, активности из курсов (**не отображает изображения из динамических страниц**), внешний объект (перечень поддерживаемых сервисов тут [https://www.learnhouse.app/ru/integrations](https://www.learnhouse.app/ru/integrations)), веб-страницы.  
![](file:///C:/Users/1/AppData/Local/Temp/msohtmlclip1/01/clip_image073.jpg)

А также при совместной работе на доске отображаются указатели мыши участников.
